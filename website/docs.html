What are SAPP & Phasor? Overview

SAPP and Phasor are server-side extensions that provide advanced scripting and customization capabilities for Halo: PC and Halo: CE dedicated servers.

SAPP, developed and maintained by sehé (halo.isimaginary.com), is the most feature-rich and widely used extension. It provides powerful Lua scripting support, anti-cheat tools, event hooks, command handling, player management, logging, and numerous under-the-hood features.

Phasor was an earlier server extension with similar goals. Though no longer actively maintained, it remains compatible with many scripts and still sees use in some legacy setups.

Installation Guide

  1. Install Halo: PC or CE (requires a valid CD key). You can find installers and patches in the README.
  2. Download SAPP from the official SAPP website. Extract the contents into your Halo server directory (where haloceded.exe or halopc.exe resides).
  3. Enable SAPP - SAPP loads automatically when strings.dll is present. If it doesn’t load, use the console command \load. To verify SAPP is running, type \version - it should show the SAPP version.
  4. Enable Lua scripting - Add lua 1 to your init.txt file (located in the server folder) or type \lua 1 in the server console. Without this, Lua scripts will not run.
  5. Download a script from this repository (e.g., gun_game.lua).
  6. Place the .lua file in your server’s lua folder (create it if necessary).
  7. Load the script using \lua_load <scriptname> (without the .lua extension). For automatic loading on server start, add lua_load <scriptname> to your init.txt.

For detailed instructions, see the Docs README or the official SAPP documentation.

Using Scripts - Quick Tips

  • Lua scripting must be enabled first. Add lua 1 to your init.txt or type \lua 1 in the server console.
  • To load a script, use \lua_load <filename> (without the .lua extension). For example: \lua_load gun_game.
  • To unload a script, use \lua_unload <filename>.
  • Scripts can be automatically loaded on server start by adding lua_load <filename> to your init.txt file.
  • Many scripts have configurable settings at the top of the .lua file. Edit them with a text editor before loading.
  • If you’re unsure whether a script is loaded, you can check the server console after running \lua_load - it will show a success message.
\lua 1
\lua_load gun_game
\lua_load zombies_standard
